The automation regenerates Sentinel-derived regional GeoTIFF sample products, verifies buyer ZIPs, uploads refreshed ZIPs to Gumroad, updates the public catalog, and submits the catalog URL to search indexing.

## Important scope and legal notes

- Source imagery: Copernicus Sentinel-2 L2A via Microsoft Planetary Computer STAC.
- AI-assisted automation is involved in selecting, processing, collating, packaging, and publishing these datasets; they are not checked by a human operator before publication.
- Products are regional sample AOI datasets, not full-planet, full-country, official, regulatory, legal, insurance, planning, financial, or agricultural-advisory datasets.
- Do not replace this source posture with scraped contacts, business lists, proprietary maps, personal data, trademarked datasets, or datasets with unclear commercial resale terms.

## If the automation reports a failure

Use the failure report generated by the weekly watchdog first. It includes the log tail, likely cause, and click-by-click recovery steps. General recovery path:

1. Read the failure heading and the first `Likely cause` line.
2. If it mentions Gumroad, open Gumroad in a browser, check that the IndexRidge products are published, and confirm payout/payment setup is complete.
3. If it mentions GitHub Pages, open the GitHub repository, go to Settings → Pages, confirm the source is `Deploy from a branch`, branch `main`, folder `/`, then save and wait for the Pages build to complete.
4. If it mentions Sentinel/STAC/network data access, wait 10–30 minutes and rerun. If the same AOI keeps failing, use a smaller or clearer AOI and keep the product described as a regional sample.
5. If it mentions PII/sensitive text, do not upload or publish the package. Remove the flagged text from generated README/report/public copy and rerun the PII scan.
6. If it mentions Python dependencies, rebuild the virtual environment and install the project requirements before rerunning.
7. After fixing, rerun the catalog manager and verify every Gumroad product URL returns HTTP 200, has a file, and is not marked unavailable.

## Safe manual checklist before publishing new products

- Product title says `sample` or `regional sample` where appropriate.
- Product copy does not claim official status, guaranteed accuracy, or full global coverage.
- Buyer ZIP contains GeoTIFF, report JSON, README, and disclaimer.
- Buyer ZIP README, disclaimer, Gumroad copy, and public catalog show the AI-assisted collation/no-human-operator-review disclosure.
- ZIP text files contain no personal names, private contact details, local filesystem paths, AWS account IDs, IAM ARNs, or secret tokens.
- Gumroad product is published, priced, and has at least one file.
- Landing page links use the API-verified Gumroad product URLs.
